# Ansible Wireguard Fedora

Setup wireguard on Fedora Server (plain package, not containerized).
Will likely work on any other RHEL derivative as well.
What is does:

- Install wireguard
- Configure ip forwarding
- Configure server and peers
- Add firewalld hooks
- Generate peer configs

## Variables

```yaml
# === Server Config
# the public hostname of the server
wireguard_hostname: wg.example.com
# port the server will listen on
wireguard_port: 51820
# server private key
wireguard_privkey: XXX
# server public key
wireguard_pubkey: XXX
# internal server ip(s) and subnet
wireguard_address: 192.168.10.1/24, ....

# === Client config
# The IPs the client can access (split tunnel or full tunnel)
wireguard_allowed_ips: 192.168.1.0/24, ...
# The DNS to use by the VPN clients
wireguard_dns: 192.168.1.1, ...

# === Peers
# Where to write the peer configurations
wireguard_peer_configs: /etc/wireguard/peers
# The peers to add to the server
wireguard_peers:
  - name: Optional peer name
    pubkey: PEER_PUBLIC_KEY
    psk: OPTIONAL_PEER_PSK
    # client ip(s) of the wireguard peer
    ips: 192.168.10.2/32, ...
```

## Adding peers

Use the following command (wireguard tools must be installed) to generate public/private key and PSK (optional).

```bash
# required: public and private key
wg genkey | tee priv.key | wg pubkey > pub.key
# optional: preshared key
wg genpsk
````

Add a new peer to `wireguard_peers` using the content of `pub.key` as `PEER_PUBLIC_KEY`.
Optionally, add the psk value generated by `wg genpsk` (improved security).
Then, assign the static IP(s) to the peer.

After the playbook run, peer configs are generated in the `wireguard_peer_configs` directory.
Copy the config for your peer and manually add the private key to connect to the server.

## Execution

The sample `playbook.yml` shows a really simple example of how to use this role.
If you already have an inventory in another repository, you can run `ansible-playbook -i inventory.yml  /path/to/ansible-wireguard-fedora/playbook.yml` to quickly execute the playbook.
A cleaner approach is to install this role using ansible-galaxy or to clone this git repo in the local ansible roles path under a simpler name like `pfruck.wireguard`.
